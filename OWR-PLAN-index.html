  • This took over the Beginning of Period protocol (which was used in the past for lumping stuff together for a week). It is now the startup of a new project in an existing notebook. It applies to the root as there is protocol here for starting a new capability.
  • There is a basic conflict in this protocol. It starts with the assumption that a new project is the same as a new notebook. This is not so. A notebook can hold a series of projects related in that it wasn't necessary to continue immediately to the next project quite often because of the lack of information what the follow on was to be and so how it was to be planned but the later project requires that the previous project be done. Obviously there is a dynamic tension here between the use of project and the use of task. The tasks that compose a project can be planned at the time the project is created.
  • When a new project is created as a singleton, the configuration information is stored in DICT BIBLIO JAY PLAN OWR as an .html file with the project name. It can then be used in turn by scripts needed for additional protocols such as adding to the biblio index and system catalog of authors. [Rework this as it is wrong in the details.]

  • This is similar to but distinct from the New Document protocol.
